# /mna-diligence — Scaevola Walks the Data Room

Run `$ARGUMENTS` through the **mna-counsel** agent (Scaevola) per
`mna-diligence`.

## Procedure

1. **Deal frame** — structure, parties, consideration, jurisdictions,
   regulatory approvals in play (Table VIII).
2. **Workstream design** — issue the checklists scoped to the deal;
   assign specialist slices:
   - contracts (change-of-control pull!) → `contract-counsel` (Gaius)
   - IP portfolio + OSS BOM → `ip-counsel` (Minerva)
   - employment/retention/parachutes → `employment-counsel` (Paulus)
   - data/privacy exposure → `privacy-counsel` (Angerona)
   - regulatory/licenses/merger-control thresholds → `regulatory-counsel`
     (Ulpian); sanctions screening → `export-controls` (Janus)
   - disputes → `litigation-counsel` (Cicero)
   - tax structure flags → `tax-counsel` (Modestinus) — every quantified
     conclusion `[CONFIRM WITH TAX ADVISOR OF RECORD]`
3. **Walk the room** — index everything reviewed; `[NOT IN DATA ROOM]`
   gap list is a first-class output; sampling rules stated.
4. **Red-flag report** — severity-ranked, each flag ending in a
   consequence: price / indemnity / condition / walk-away.
5. **R&W heatmap + closing checklist** — per the skill formats.
6. **Verification** — through Tribonian.
7. **Gates & escalation** — citation-integrity@1, aba-512-ethics@1;
   deal-commitment recommendations are Tribune's-Veto matters (authority
   is human); CLO packet prepared for materiality thresholds.
8. **File** — Diligence Report Format to `output/mna/<topic>-<date>.md`.

## Output contract

Privilege banner · deal frame · workstream status table · data-room index
+ gap list · red flags with consequences · R&W heatmap · closing
checklist · specialist appendices · authorities.
