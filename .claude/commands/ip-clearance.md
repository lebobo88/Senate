# /ip-clearance — Minerva Searches Before She Clears

Run `$ARGUMENTS` through the **ip-counsel** agent (Minerva) per
`ip-clearance`.

## Procedure

1. **Define the thing being cleared** — precisely, element by element.
2. **Select the protocol** by type (or infer):
   - `fto` — claim-mapping structure + search specification
   - `trademark` — the 5-step clearance ladder, by class and jurisdiction
   - `oss` — compatibility matrix + copyleft-boundary analysis +
     obligations inventory
   - `copyright` — ownership/assignment chain review
   - `asset` — creative-asset clearance (Garland interface: marks in
     frame, releases, stock-license scope)
3. **Search or specify** — run what can be run against provided materials;
   where live databases are required, output the **search specification**
   and mark `[SEARCH NOT PERFORMED — clearance unavailable]`. Never soft-
   clear (Phase-1 limits).
4. **Analyze** — claim charts / mark comparisons / license-boundary
   analysis as applicable. AGPL-in-SaaS is a Critical flag.
5. **Specialist pulls** — license terms in contracts → `contract-counsel`
   (Gaius); IP reps in deals → `mna-counsel` (Scaevola).
6. **Verification** — through Tribonian; license texts in the matter are
   `[VERIFIED:source-in-matter]`.
7. **Gates & verdict** — open-source-license-compatibility@1 where OSS in
   scope + citation-integrity@1. Verdict: cleared / cleared-with-conditions
   / not-cleared / search-required — confidence attached. Unclear rights ⇒
   gatekeeper halt + Tribune's Veto.
8. **File** — Clearance Report to `output/ip/<topic>-<date>.md`.

## Output contract

Privilege banner · what-was-cleared definition · search record or
specification · findings (charts/matrices) · obligations inventory (OSS) ·
verdict with confidence · injunction-risk note · authorities.
