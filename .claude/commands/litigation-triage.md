# /litigation-triage — Cicero Takes the Case Apart

Run `$ARGUMENTS` through the **litigation-counsel** agent (Cicero) per
`litigation-playbook`.

## Procedure

1. **Clocks first** — limitation periods, response deadlines, notice
   windows (contractual + insurer). Unknown ⇒ `[DEADLINE UNCONFIRMED —
   calendar with local counsel]`, at the top of the artifact.
2. **Hold trigger** — litigation reasonably anticipated? ⇒ draft the hold
   notice (skeleton in the skill) and flag custodians/systems.
3. **Forum & law** — where, what law, arbitration clause check
   (Table VIII).
4. **Elements vs. evidence** — full table per claim/defense; gaps become
   the investigation plan.
5. **Steel-man** — the opponent's best case, written as their best lawyer
   would (mandatory).
6. **Procedural map** — posture + next 3 decision points with
   cost/time/probability-shift each.
7. **Settlement tree** — EV math shown, probability sources named
   (the elements table, not optimism), BATNA stated.
8. **Specialist pulls** — contract-interpretation slices → Gaius;
   employment disputes → Paulus; privileged-data handling → Angerona.
9. **Verification** — hardest pass: every authority through Tribonian
   (a fabricated case citation is the canonical catastrophe — Table III).
10. **Gates & file** — privilege-handling@1, citation-integrity@1,
    aba-512-ethics@1. Settlement recommendations carry the Tribune's-Veto
    marker (authority is human). Case Assessment Format to
    `output/litigation/<topic>-<date>.md` with work-product banner
    expressly invoking anticipation of litigation.

## Output contract

Work-product banner · **deadlines table first** · hold status ·
elements-vs-evidence · both-sides analysis · procedural map · settlement
tree with assumptions · recommended posture · authorities.
